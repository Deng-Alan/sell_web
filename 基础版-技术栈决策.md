# 基础版 MVP 技术栈决策

## 1. 决策概述

本文档记录基础版商品展示网站当前确认的技术选型决策。

本版本决策覆盖并替代之前的 `Vue/Nuxt + Express + MySQL` 规划，当前统一采用：
- 前端：Next.js
- 后端：Spring Boot
- 数据库：PostgreSQL
- 认证：JWT

## 2. 技术栈选择

### 2.1 前端技术栈

| 技术 | 版本建议 | 选择理由 |
|---|---|---|
| Next.js | 15.x | 支持 SSR/SSG，SEO 友好，适合展示型网站 |
| React | 19.x | 生态成熟，组件化能力强 |
| TypeScript | 5.x | 类型安全，提升可维护性 |
| Tailwind CSS | 3.x / 4.x | 响应式开发效率高，适合后台和展示页 |

**选择理由：**
- Next.js 适合需要 SEO 的商品展示站
- App Router 结构清晰，便于前台和后台页面拆分
- React 生态完善，适合后续扩展
- Tailwind CSS 适合快速构建前后台界面

### 2.2 后端技术栈

| 技术 | 版本建议 | 选择理由 |
|---|---|---|
| Java | 21 LTS | 长期支持版本，适合 Spring Boot 3.x |
| Spring Boot | 3.3.x / 3.4.x | 企业级框架，适合后台管理和 API 服务 |
| Spring Web | - | 提供 REST API |
| Spring Security | - | 适合接入 JWT 认证 |
| Spring Validation | - | 请求参数校验 |
| JWT | - | 管理员登录与接口认证 |

**选择理由：**
- Spring Boot 更适合规范化后台和后续扩展
- Spring Security + JWT 是成熟组合
- Java 生态在后台管理、权限和数据层更稳定

### 2.3 数据库

| 技术 | 版本建议 | 选择理由 |
|---|---|---|
| PostgreSQL | 16.x / 17.x | 稳定、功能完善、适合结构化业务数据 |
| pgAdmin / DBeaver | - | 数据库管理方便 |

**选择理由：**
- 商品、分类、联系方式、配置都属于典型关系型数据
- PostgreSQL 在约束、索引和 JSON 扩展能力上更强
- 适合后续订单、用户、审计日志等扩展

### 2.4 认证方案

| 技术 | 用途 |
|---|---|
| JWT | 管理员登录态与接口访问认证 |
| Spring Security | 认证与鉴权入口 |
| BCrypt | 密码哈希 |

**选择理由：**
- 当前只需要管理员后台登录，不需要复杂用户体系
- JWT 无状态实现简单，适合前后端分离

### 2.5 部署方案

| 技术 | 选择理由 |
|---|---|
| Nginx | 反向代理、静态资源分发 |
| Next.js Node Runtime | 前端 SSR 服务 |
| Spring Boot Jar | 后端 API 服务 |
| PostgreSQL | 关系型数据库服务 |
| systemd / Docker / 宝塔 | 视服务器环境选择进程托管方式 |

**部署结构：**
```text
用户请求 -> Nginx
         -> Next.js
         -> Spring Boot API
         -> PostgreSQL
```

## 3. 项目结构原则

### 3.1 前端
- 使用 Next.js App Router
- 前台页面与后台页面共用一个前端工程
- 后台通过独立路由段管理，如 `/admin`

### 3.2 后端
- 使用 Spring Boot 提供 REST API
- 按 `controller / service / repository / entity / dto` 分层
- JWT 登录只覆盖管理员后台

### 3.3 数据层
- 使用 PostgreSQL
- 使用 SQL migration 或 Flyway/Liquibase 管理迁移
- 所有核心表结构以关系模型为主

## 5. 非功能需求实现

| 需求 | 实现方式 |
|---|---|
| SEO | Next.js SSR / Metadata API |
| 移动端适配 | Tailwind CSS 响应式布局 |
| 图片上传 | Spring Boot Multipart + 本地或对象存储 |
| 表单校验 | 前端 Zod/React Hook Form，后端 Spring Validation |
| 后台认证 | Spring Security + JWT |

## 6. 二期扩展预留

| 扩展功能 | 技术方案 |
|---|---|
| 在线支付 | Spring Boot 对接支付网关 |
| 订单系统 | PostgreSQL 新增订单表与状态流转 |
| 自动发货 | 后端新增库存池与发货服务 |
| 用户系统 | Spring Security 扩展用户认证体系 |

## 7. 决策依据

本技术栈选择基于以下考虑：
1. **前端要求明确**：指定使用 Next.js
2. **后端要求明确**：指定使用 Spring Boot + JWT
3. **数据库要求明确**：指定使用 PostgreSQL
4. **SEO 需求**：Next.js 适合商品展示站
5. **后续扩展**：Spring Boot + PostgreSQL 更适合后台能力扩展

## 8. 风险与注意事项

| 风险 | 说明 |
|---|---|
| 技术栈切换成本 | 之前按 Nuxt/Express 规划的骨架需要废弃或重建 |
| 工程复杂度上升 | Java 后端初始化成本高于 Express |
| 前后端语言不统一 | 共享类型不能直接复用，需要 API DTO 和文档约束 |

## 9. 当前结论

当前正式技术基线为：
- 前端：Next.js + React + TypeScript + Tailwind CSS
- 后端：Spring Boot + Spring Security + JWT
- 数据库：PostgreSQL
- 部署：Nginx + Next.js + Spring Boot + PostgreSQL
