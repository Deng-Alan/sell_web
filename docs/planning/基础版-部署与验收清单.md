# 基础版部署与验收清单

## 1. 目标

本文档用于 feature-013 的本地联调、部署前检查和验收确认。范围仅覆盖当前基础版 MVP 的可执行步骤，不涉及业务扩展功能。

## 2. 本地环境要求

- Node.js 18+
- Java 17+
- Maven 3.9+
- PostgreSQL 15+
- `psql` 命令行工具可用

## 3. 启动顺序

### 3.1 创建数据库

默认数据库名为 `sell_web`。如果本地还没有创建，可先执行：

```bash
createdb -U postgres sell_web
```

如果不方便使用 `createdb`，也可以在 `psql` 中手动执行：

```sql
CREATE DATABASE sell_web;
```

### 3.2 启动后端

后端默认读取以下环境变量：

- `APP_PORT`，默认 `8080`
- `DB_HOST`，默认 `localhost`
- `DB_PORT`，默认 `5432`
- `DB_NAME`，默认 `sell_web`
- `DB_USER`，默认 `postgres`
- `DB_PASSWORD`，默认 `postgres`
- `JWT_SECRET`，默认 `change-this-secret`

启动命令：

```bash
mvn -f backend/pom.xml spring-boot:run
```

后端启动后，Flyway 会自动执行 `database/migrations/` 对应的迁移脚本。

健康检查接口：

```text
GET http://localhost:8080/api/health
```

预期返回：

```json
{"success":true,"message":"ok"}
```

### 3.3 启动前端

前端默认启动在 `http://localhost:3000`。

启动命令：

```bash
npm run dev:frontend
```

也可以直接在前端目录执行：

```bash
npm --prefix frontend run dev
```

## 4. 数据库迁移验收

- 确认 `database/migrations/` 和 `backend/src/main/resources/db/migration/` 的迁移版本号一致
- 确认后端启动时日志中出现 Flyway 迁移执行信息
- 确认业务表已创建，包括管理员、分类、商品、图片、联系方式、站点配置和首页区块

## 5. 默认管理员

本地开发和首次部署使用默认管理员：

- 用户名：`admin`
- 密码：`admin123`

说明：

- 该账号仅用于本地开发或首次部署引导
- 生产环境上线前必须修改密码或删除该记录
- 后端登录接口为 `POST /api/auth/login`

## 6. 公共接口

前台只读接口统一挂在 `/api/public/**`，默认允许匿名访问。

接口清单：

- `GET /api/public/home-config`
- `GET /api/public/categories`
- `GET /api/public/contacts`
- `GET /api/public/products`
- `GET /api/public/products/{id}`

辅助接口：

- `GET /api/health`

## 7. 前端构建验收

执行以下命令验证前端打包：

```bash
npm run build:frontend
```

也可以在前端目录执行：

```bash
npm --prefix frontend run build
```

验收通过标准：

- 构建成功，无阻断性错误
- 首页、商品列表页、商品详情页能正常参与生产构建
- 后台管理页不影响前台构建

## 8. 联调验收清单

- 后端 `GET /api/health` 正常返回 `ok`
- 前端首页可打开并展示首屏、模块说明和联系承接区
- 前台商品列表页可打开并展示筛选、列表和咨询入口
- 前台商品详情页可打开并展示详情、联系入口和 SEO 信息
- 管理员可用 `admin / admin123` 完成登录
- 公共接口可在未登录状态下读取首页配置、分类、商品和联系方式

## 9. 已知非阻塞问题

- Next.js 生产构建会提示多 `package-lock.json` 的 workspace root 警告
- 默认管理员密码仅适用于本地开发，生产环境必须立即修改
- 公共接口当前只负责读取，不包含下单、支付或自动发货

## 10. 建议执行顺序

1. 创建数据库 `sell_web`
2. 启动 PostgreSQL
3. 启动后端并确认 `/api/health`
4. 启动前端并确认首页、列表页、详情页
5. 执行前端构建验收
6. 使用默认管理员登录后台
7. 检查公共接口和页面联通情况
